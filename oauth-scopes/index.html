OAuth Scopes

Warning

This page is under construction, I'll remove it when the page is g2g.

This document outlines the OAuth scopes requested by Clairify’s AI summarization platform to ensure clarity, transparency, and alignment with the Principle of Least Privilege. It is intended for IT administrators, CISOs, and cybersecurity professionals evaluating Clairify in regulated or sensitive environments.

2. Our Permission Philosophy

  • Clairify requests only the minimum OAuth scopes required for functionality.
  • All access is read-only. Clairify does not request permission to send, delete, or modify emails.
  • Clairify can access attachments, but not drafts or calendar information.
  • We never request elevated scopes without justification and re-authorization.

3. OAuth Consent Flow Overview

During onboarding, users are prompted to grant permissions via Google’s OAuth 2.0 consent screen. Tokens are time-limited and securely stored. The flow includes:

  • Google account selection
  • Explicit user consent
  • Redirection to Clairify upon approval
  • Token acquisition with read-only scope

4. OAuth Scopes Requested

Non-Sensitive Scopes

  • openid – Required for identity verification.
  • https://www.googleapis.com/auth/userinfo.email – Access to the user’s email address for account linking.
  • https://www.googleapis.com/auth/userinfo.profile – Basic profile information used for personalized UI fourishes.

Restricted Scope

  • https://www.googleapis.com/auth/gmail.readonly
  • Purpose: Allows Clairify to access Gmail message contents strictly for summarization.
  • Permissions: Read-only access to message subject lines, headers, and body content. No send, modify, or delete capabilities.
  • Security: Content is processed in memory and encrypted at rest.

Note

At this time we have an administrative view that allows our engineering team to decrypt and view email.

5. Scope Change Policy

  • Any scope addition requires renewed user consent.

6. Token Lifecycle & Revocation

  • OAuth tokens are one-time use and sessions are automatically refreshed using secure methods.
  • Customers can revoke access at any time via:
  • The Google Account Security Settings dashboard
  • Enterprise Admin tools (e.g., Google Workspace Admin)
  • Clairify automatically expires sessions on inactivity or user logout.

7. Security Considerations

  • Tokens are securely stored by encrypting at rest.
  • All OAuth interactions are logged for auditing.
