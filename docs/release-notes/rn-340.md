### Text-to-Speech: On-Demand Email Audio

Tap any email card to hear it read aloud. If the audio is already cached it plays immediately — otherwise it's generated on the spot. No more reading every card; just listen.

### Text-to-Speech: 24-Hour Email Playlist

Emails received in the last 24 hours are now available as a continuous audio playlist. Think of it as a podcast version of your inbox — listen through without touching the screen.

### Text-to-Speech: Automatic Briefing Audio

Morning and evening briefings are now automatically converted to audio so they're ready when you open the app. Consume your briefings hands-free during commutes, between meetings, or while multitasking.

### Text-to-Speech: Category-Filtered Playlists

Choose which categories or message groups — Important, Finance, Product, Sales, or your custom groups — are included in your audio playlist. Listen to what matters, skip the rest.

### Text-to-Speech: Playback Improvements

- Audio playlists now include pause and resume controls during continuous playback.
- A visual indicator shows which item is currently playing in the playlist.
- Played and unplayed items are visually distinguished so you can track what you've listened to.
- The dashboard now displays unplayed audiobook item counts for quick visibility.
- On-demand playlist generation is now a premium feature.

### PDF Attachments (Outlook)

You can now open PDF attachments directly from Clairify when connected to an Outlook account. For emails where the body is just "See attached," you no longer need to leave the app.

### Chat Interface (Preview)

A proof-of-concept chat interface is now available in the mobile app. Starting with a Q&A agent against Clairify documentation, this is the first step toward conversational interaction with your executive information.

### Magic Button on Briefing Cards

The magic button is now enabled on briefing cards, giving you quick access to actions like "Get Latest" without extra navigation. Previously, briefing cards had no quick-action surface.

### Briefing Card Swiping

Briefing cards in the briefing channel now support standard swiping behavior in all directions. Swipe left to discard, swipe right to mark as read and send to the back of the stack. Unread and read states are indicated with envelope icons.

### Briefing Card Mark-as-Read and Discard

New backend endpoints support mark-as-read and discard actions for briefing reports. The briefing list now shows only unread reports by default.

### Onboarding Channel Improvements

- Onboarding cards now support free swiping in all directions — left for previous, down or right for next.
- The magic button is no longer activated on onboarding cards where it doesn't apply.
- Onboarding cards now display badge numbers for consistent UI alignment.

### Generate Briefing Card Refinement

The generate briefing card UI has been cleaned up — the duplicate "Generate Briefing" element is removed and the empty-state experience is refined.

### Account Sync UX

Syncing an account now shows an in-progress loading state with informative status messages from the backend, instead of exiting immediately with no feedback.

### Standardized Logout

The app now consistently uses "Log Out" everywhere. The redundant "Sign Out" option has been removed from the account selection long-press menu, which was misleading users into thinking they could selectively disconnect individual email accounts.

### OpenCLAW Integration (Admin)

The web admin panel now includes configuration controls for connecting an OpenCLAW instance to an organization, with per-member access management.

### TTS Admin Controls

Text-to-speech settings are now configurable from the web admin panel.

- **Fixed:** AI engine display issues across multiple web admin pages, including missing engines like Google Gemini 2.5 Flash Lite.
- **Fixed:** TTS category filter selection bugs across admin server and webserver.
